# 股池管理工具 PRD

最后更新：2026-05-17  
当前阶段：个人股池管理 + 分享/关注能力原型  
线上地址：`https://simonlichaooooo.github.io/stock-pool-codex/`

## 1. 产品目标

本产品的目标是构建一个面向个人投资研究者的股池管理与观点分享工具。

用户可以在一个统一工具里完成股票录入、估值假设维护、预期回报测算、个人研究记录、观点发布和关注他人股池等动作。产品希望解决的问题不是实时交易，而是帮助用户持续记录和复盘自己对公司的判断。

核心目标：

- 管理个人关注股票，覆盖 A 股、港股、美股。
- 记录关键估值假设，包括市值、前瞻利润、前瞻 PE、净现金、股东回报等。
- 自动计算前瞻市值、Upside、Forward PE、前瞻股东回报率。
- 区分私有记录和公开观点，用户可以选择只保存自己看，或发布为可分享的投资观点快照。
- 通过版本记录追踪投资观点变化。
- 支持关注其他投资者，查看、复制或订阅他人分享的股票观点。
- 提供公开广场，帮助用户发现公开分享的研究者。

长期目标是从个人股池工具逐步演进为一个轻量的投资研究网络：每个用户都可以维护自己的研究框架，也可以关注他人的公开观点变化。

## 2. 用户类型

### 2.1 个人投资者

主要用户。  
使用产品维护自己的股票观察池、记录估值假设、跟踪预期回报。

典型诉求：

- 快速录入关注股票。
- 保存自己的盈利、估值和现金调整假设。
- 看到当前假设下的 Upside 和 Forward PE。
- 后续能回看自己当时为什么这么判断。

### 2.2 投资内容创作者 / 研究型用户

愿意将部分股池和观点分享给粉丝或朋友的用户。

典型诉求：

- 选择性发布部分持仓或关注股票。
- 每次发布时记录变更说明。
- 能把分享链接发给别人。
- 让关注者看到自己的最新公开观点。
- 未来可能希望通过订阅、私域或付费方式变现。

### 2.3 关注者 / 粉丝

通过分享链接或广场发现某位发布者，并关注其股池。

典型诉求：

- 查看发布者公开或私密分享的股池。
- 一键复制某只股票到自己的股池中继续研究。
- 订阅某只股票，后续跟随发布者更新。
- 同时维护自己的独立判断。

### 2.4 管理员

平台管理者，目前仅项目 owner。

典型诉求：

- 查看用户列表。
- 查看用户公开和私有股池。
- 管理广场展示。
- 禁用违规用户分享能力。
- 隐藏不适合展示的股票或用户。

## 3. 核心功能

### 3.1 登录与账户

- 邮箱 Magic Link 登录。
- GitHub OAuth 登录。
- 首次登录后设置唯一昵称。
- 登录后保存 session，过期前自动刷新。

### 3.2 个人股池管理

- 新增股票。
- 编辑股票。
- 删除股票。
- 股票名称或代码搜索匹配。
- 自动识别股票名称、代码、市场、交易所。
- 自动获取最新市值。
- 支持股票置顶、置底和排序。
- 支持搜索和筛选。

### 3.3 估值假设录入

- 当前市值。
- 货币单位：`CNY`、`HKD`、`USD`。
- 前瞻年份。
- 前瞻利润。
- 前瞻 PE。
- 利润备注。
- 前瞻股东回报。
- 股东回报政策。
- 净现金盘点。
- 净现金口径。
- 净现金折扣系数。
- 个人研究备注。

### 3.4 自动测算

- 前瞻市值。
- Upside。
- Forward PE。
- 前瞻股东回报率。
- 港股支持 `20%股息税后` 口径。

### 3.5 保存与发布

- 保存：只更新自己的私有记录，不生成发布版本。
- 发布：保存当前内容，并发布到分享股池，生成版本快照。
- 发布时可以填写变更说明。
- 已发布股票在列表中显示分享状态和上次发布时间。
- 已发布股票可以查看发布历史，历史展示每次发布时的市值、前瞻利润、PE、净现金、股东回报、计算结果和变更说明。
- 私人保存不会生成发布历史；对外展示和订阅同步以最新发布快照为准。

### 3.6 分享与关注

- 用户可以复制自己的分享链接。
- 访问分享链接后，登录用户可以关注发布者。
- 用户可以在顶部筛选自己股池或已关注用户股池。
- 查看关注人股池时支持：
  - 一键复制：复制到自己股池，之后可自由修改。
  - 订阅：复制到自己股池，并后续跟随发布者更新。

### 3.7 广场

- 展示公开分享的发布者。
- 展示发布者昵称、已发布股票数量、最近发布时间、代表股票。
- 支持搜索和排序。
- 可以进入发布者股池。

### 3.8 管理后台

- 仅管理员可见。
- 查看用户。
- 查看用户股池，并区分私有记录和公开发布。
- 隐藏或恢复广场用户。
- 禁用或恢复用户分享能力。
- 隐藏或恢复单只股票。
- 记录管理员操作日志。

## 7. 产品设计原则

### 7.1 研究优先，而不是交易优先

产品定位是投资研究和观点记录工具，不是行情交易软件。  
页面应突出估值假设、预期回报、观点变化和研究笔记，而不是短线价格波动。

### 7.2 私有记录和公开观点严格分离

保存和发布是两个不同动作：

- 保存用于个人研究记录。
- 发布用于分享观点并生成版本快照。

用户必须清楚知道哪些内容只有自己可见，哪些内容会被别人看到。

### 7.3 观点要可追踪

发布不是覆盖旧观点，而是生成新版本。  
长期应该能看到一个投资者对同一只股票的观点如何变化。

### 7.4 复制和订阅要区分

一键复制表示用户拿走一份独立副本，后续可以自由修改。  
订阅表示用户愿意继续跟随发布者更新，订阅记录不应被随意修改。

### 7.5 金融工具感

视觉应克制、专业、信息密度适中。  
避免 demo 感、过大圆角、过亮颜色、过松散间距和过多装饰。

### 7.6 合规提示要持续存在

产品中展示的内容是用户个人研究记录，不构成投资建议。  
公开分享、广场、发布者主页等位置都应有弱化但明确的合规提示。

## 8. 当前限制

### 8.1 技术限制

- 当前是单文件静态应用，所有逻辑集中在 `index.html`。
- 没有前端框架。
- 没有 TypeScript。
- 没有自动化测试。
- 没有构建流程。
- 没有后端业务服务。
- 数据库迁移需要手动执行 SQL。

### 8.2 数据限制

- 行情接口依赖第三方公开接口，稳定性不可完全保证。
- 部分股票可能搜索不到。
- 市值口径可能与专业终端存在差异。
- 汇率依赖公开免费 API，并使用内置备用汇率兜底。

### 8.3 产品限制

- 付费订阅尚未实现。
- 发布者隐私和内容审核机制还比较基础。
- 广场推荐机制较简单。
- 订阅同步是加载时前端合并，不是实时后端推送。
- 管理员能力可用但不够精细。

### 8.4 安全和权限限制

- 管理员识别依赖 GitHub 用户名和 `admin_users` 表。
- 前端仍包含 Supabase anon key，虽然符合 Supabase 客户端使用模式，但所有访问控制必须依赖 RLS。
- 行情 API 直接由浏览器访问，无法隐藏请求来源或做稳定缓存。

## 9. 商业化方向（如果有）

当前产品尚未接入商业化能力，但未来存在以下方向：

### 9.1 付费订阅发布者

发布者可以设置：

- 免费公开股池。
- 私密链接股池。
- 付费订阅股池。
- 分层订阅，例如普通观点、高频更新、深度备注。

平台可以从订阅收入中抽成。

### 9.2 投资研究者主页

为优质发布者提供个人主页，展示：

- 股池表现。
- 历史观点变化。
- 关注人数。
- 代表性研究。
- 公开组合。

长期可以形成投资研究者的个人品牌页。

### 9.3 专业版工具

面向重度用户提供 Pro 功能：

- 更多股池数量。
- 更多历史版本。
- 高级筛选。
- 数据导出。
- 更稳定行情源。
- 指标模板。
- 多组合管理。
- 价格提醒和观点提醒。

### 9.4 数据服务和机构版

如果未来用户规模扩大，可以沉淀：

- 用户共识预期。
- 热门股票观点变化。
- 发布者关注热度。
- 股池持仓分布。

但此方向涉及较高合规要求，需要谨慎推进。

## 10. 未来扩展方向

### 10.1 产品扩展

- 组合视图：从单只股票扩展到组合层面。
- 组合权重：记录仓位和目标仓位。
- 收益追踪：记录观点发布后的表现。
- 观点复盘：按季度或财报节点复盘。
- 财报日历：提醒重点股票财报。
- Watchlist 标签：按行业、主题、风险等级分组。
- 多估值情景：乐观、中性、悲观三套假设。
- 敏感性分析：利润和 PE 变化对 Upside 的影响。

### 10.2 社交扩展

- 发布者关注动态流。
- 股票观点评论。
- 私密分享群组。
- 付费订阅。
- 投资者排行榜。
- 精选研究广场。

### 10.3 技术扩展

- 从单文件重构为 React / Next.js 或类似现代前端架构。
- 引入组件库和设计系统。
- 引入 TypeScript。
- 引入 Supabase Edge Functions 或独立后端。
- 行情和汇率服务端缓存。
- 自动化测试。
- 数据库迁移工具。
- CI/CD。

### 10.4 合规扩展

- 更明确的免责声明。
- 发布内容审核。
- 举报机制。
- 付费内容合规审核。
- 用户协议和隐私政策。
- 投资建议边界说明。
