# 股池管理需求文档

## 1. 产品概述

股池管理是一个面向个人投资研究的 Web 应用，用于保存股票池、维护估值假设、计算预期回报，并支持将个人观点以公开或私密方式分享给其他登录用户。系统当前为单页前端应用，后端依赖 Supabase Auth、PostgREST、RLS 策略和数据库表。

## 2. 用户角色

| 角色 | 说明 | 核心权限 |
| --- | --- | --- |
| 未登录用户 | 访问应用但未认证 | 查看登录页，使用 GitHub 或邮箱 Magic Link 登录 |
| 普通用户 | 已登录且完成昵称设置 | 管理自己的股池、发布分享、关注他人、复制或订阅他人股票 |
| 发布者 | 普通用户的一种状态，已发布股票或开启公开分享 | 被关注、被广场展示、通过分享链接传播股池 |
| 管理员 | GitHub 用户名命中 `admin_users` 表 | 查看用户和股票记录，隐藏用户/股票，禁用分享，维护管理备注 |

## 3. 模块需求

### 3.1 认证与用户资料模块

**目标**

保证用户身份可信，并为分享、关注、管理等社交功能提供唯一用户资料。

**功能需求**

- 支持 GitHub OAuth 登录。
- 支持邮箱 Magic Link 登录。
- 登录会话需保存到浏览器本地，并在 token 即将过期时自动刷新。
- 首次登录用户必须设置唯一昵称后才能进入主应用。
- 用户资料需包含邮箱、昵称、分享可见性、广场隐藏状态、分享禁用状态、管理员备注等字段。
- 昵称必须唯一，重复时给出明确提示。
- 用户可退出登录，退出后清空本地会话和当前页面状态。

**数据依赖**

- `auth.users`
- `profiles`
- 本地存储键：`stockpool.supabaseSession`

**权限要求**

- 用户只能创建和更新自己的 profile。
- 已登录用户可读取其他用户 profile，用于关注、广场和分享展示。
- 管理员可更新任意用户的管理字段。

### 3.2 我的股池模块

**目标**

让用户维护自己的股票研究列表，并以统一表格展示关键估值指标。

**功能需求**

- 支持新增、编辑、删除股票。
- 支持按股票名称或代码搜索股池记录。
- 支持按市场筛选：全部、A 股、H 股、美股。
- 支持按分享状态筛选已分享股票。
- 支持股票置顶、置底和手动顺序排序。
- 表格需展示：
  - 股票名称与代码
  - 最新市值
  - upside
  - Forward PE
  - 前瞻股东回报率
  - 前瞻市值
  - 前瞻利润
  - 前瞻 PE
  - 净现金
  - 现金折扣
  - 前瞻股东回报
  - 分享状态
  - 来源信息
- 支持按 upside、Forward PE、前瞻股东回报率排序。
- H 股股东回报率支持切换“20% 股息税后”口径。

**数据依赖**

- `stock_records.payload`
- `stock_records` 的来源、分享、发布时间等冗余列

**业务规则**

- 用户只能读写自己的私有股票记录。
- 被订阅股票的核心估值字段不可直接编辑，只能维护“我的备注”。
- 删除股票前需要确认。

### 3.3 股票匹配与行情模块

**目标**

降低录入成本，自动匹配股票基础信息并获取最新市值。

**功能需求**

- 在新增/编辑抽屉中输入股票名称或代码时触发搜索。
- 优先使用东方财富搜索接口匹配 A 股、H 股、美股。
- 搜索失败或无结果时，使用内置备用样例。
- 选择股票后自动填充：
  - 名称
  - 代码
  - 市场
  - 交易所
  - 行情标识
  - 默认展示货币
  - 最新市值
  - 市值更新时间
- 市值优先使用腾讯行情接口，失败后使用东方财富行情接口，再失败则使用内置备用市值。
- 支持人民币、港币、美元展示和换算。
- 汇率优先从 `open.er-api.com` 获取，失败后使用固定备用汇率。

**业务规则**

- 所有估值计算统一以 CNY 亿为内部基准。
- 用户可切换展示货币，但保存时金额字段仍需转换为 CNY。

### 3.4 估值测算模块

**目标**

基于用户输入的前瞻利润、PE、净现金和股东回报假设，实时计算估值结果。

**输入字段**

- 前瞻利润
- 前瞻 PE
- 前瞻年份
- 利润备注
- 前瞻股东回报
- 股东回报政策
- 净现金
- 净现金口径
- 净现金打折系数
- 我的备注

**计算规则**

- 前瞻市值 = 前瞻利润 × 前瞻 PE + 净现金 × 净现金打折系数
- upside = 前瞻市值 / 最新市值 - 1
- Forward PE = (最新市值 - 净现金 × 净现金打折系数) / 前瞻利润
- 前瞻股东回报率 = 前瞻股东回报 / 最新市值
- H 股税后口径下，前瞻股东回报率 × 0.8。

**功能需求**

- 抽屉右侧实时展示测算结果。
- 输入金额字段时需按当前展示货币进行显示，并按汇率转为 CNY 保存。
- 净现金打折系数应限制在 0 到 1 的合理范围。
- 对无效或缺失数据展示 `--`，避免错误计算污染界面。

### 3.5 发布与分享模块

**目标**

允许用户把个人股池观点发布给其他用户，并保留发布版本记录。

**功能需求**

- 用户可在股票编辑抽屉中填写发布变更说明。
- 点击发布时，系统先保存股票，再将股票标记为已分享。
- 发布后记录发布时间。
- 每次发布需写入一条 `stock_publications` 版本记录。
- 支持设置分享方式：
  - 私密：仅通过分享链接/关注关系进入
  - 公开：可出现在广场
- 支持复制分享链接，链接携带发布者用户 ID。
- 分享内容需展示投资建议免责声明。
- 被管理员禁用分享的用户不能发布，也不能设置公开分享。

**数据依赖**

- `stock_records.is_shared`
- `stock_records.last_published_at`
- `stock_publications`
- `profiles.share_visibility`
- `profiles.share_banned`

**业务规则**

- 订阅来的股票不能直接发布，必须先转为自己的股票。
- 发布版本号按同一股票当前最大版本号递增。

### 3.6 关注、复制与订阅模块

**目标**

让用户可以消费他人的分享股池，并选择静态复制或动态订阅。

**功能需求**

- 通过分享链接进入时展示发布者邀请横幅。
- 用户可关注发布者。
- 关注后，发布者会出现在“我的股池”顶部下拉框中。
- 查看他人分享股池时，表格展示发布者已发布且未被管理员隐藏的股票。
- 对他人股票支持：
  - 一键复制：复制一份静态记录到自己的股池
  - 订阅：复制到自己的股池并建立订阅关系
- 订阅记录在发布者更新后，应在自己的股池中提示“有更新”。
- 支持标记所有订阅更新为已读。
- 支持取消订阅。
- 支持将订阅股票转为自己的股票。

**数据依赖**

- `follows`
- `stock_subscriptions`
- `stock_records.source_user_id`
- `stock_records.source_stock_id`
- `stock_records.source_type`
- `stock_records.source_seen_published_at`
- `stock_records.last_source_updated_at`

**业务规则**

- 同一用户不能关注自己。
- 同一用户对同一发布者的同一源股票只能订阅一次。
- 订阅股票同步发布者最新 payload，但保留本地 id、排序、置顶状态、分享状态、来源信息和我的备注。
- 源股票停止分享或不可读时，订阅记录需标记为“已停止分享”。

### 3.7 广场模块

**目标**

聚合公开分享的发布者，帮助用户发现其他人的公开股池。

**功能需求**

- 展示所有满足条件的公开发布者：
  - `share_visibility = public`
  - 未被广场隐藏
  - 未被禁用分享
  - 至少有一只已分享且未隐藏的股票
- 支持按发布者昵称搜索。
- 支持排序：
  - 最近发布
  - 发布数量
- 发布者卡片展示：
  - 昵称
  - 已发布股票数量
  - 最近发布时间
  - 最多 3 只代表股票及其 upside、Forward PE
- 点击进入发布者股池视图。

**数据依赖**

- `profiles`
- `stock_records`

### 3.8 管理后台模块

**目标**

为管理员提供基础内容治理能力，降低公开分享风险。

**功能需求**

- 仅管理员可看到“管理后台”导航。
- 管理员可查看所有用户列表。
- 用户列表需展示昵称、邮箱、分享状态、广场隐藏状态、分享禁用状态。
- 管理员可选择用户查看其股池记录。
- 管理员可执行：
  - 从广场隐藏/恢复用户
  - 禁用/恢复用户分享能力
  - 保存管理员备注
  - 隐藏/恢复某只公开股票
- 管理操作应写入审计日志。

**数据依赖**

- `admin_users`
- `profiles`
- `stock_records`
- `moderation_actions`

**权限要求**

- 管理员身份通过 GitHub 用户名与 `admin_users.github_username` 匹配。
- 管理员可读取全部股票记录。
- 管理员可更新股票的隐藏状态和用户的管理字段。

### 3.9 数据安全与权限模块

**目标**

通过 Supabase RLS 保护多用户数据边界。

**功能需求**

- `stock_records` 开启 RLS。
- 普通用户只能读写自己的股票记录。
- 已登录用户可读取已分享且未隐藏的股票记录。
- 管理员可读取和治理全部股票记录。
- `profiles` 开启 RLS。
- `stock_subscriptions` 只能由订阅者本人读写。
- `follows` 只能由关注者本人读写。
- `stock_publications` 可被登录用户读取，只有股票所有者可发布。
- `moderation_actions` 只有管理员可读写。

### 3.10 界面与响应式模块

**目标**

在桌面和移动端都能高效管理股池。

**功能需求**

- 顶部固定导航展示产品名、用户邮箱、昵称和退出按钮。
- 主导航包含股池、广场、管理后台。
- 桌面端使用宽表格展示股池。
- 移动端表格自动转为卡片式布局。
- 新增/编辑股票使用右侧抽屉。
- 抽屉内采用左侧表单、右侧结果面板布局，移动端自动单列。
- 空状态需给出明确下一步引导。
- 重要失败场景需给出可理解的错误提示。

## 4. 核心数据模型

### 4.1 stock_records

保存用户股票记录。当前采用结构化列 + `payload jsonb` 的混合设计，`payload` 保存完整业务对象，结构化列用于权限、筛选、分享和订阅查询。

关键字段：

- `id`
- `user_id`
- `payload`
- `source_user_id`
- `source_stock_id`
- `source_nickname`
- `source_type`
- `last_source_updated_at`
- `source_seen_published_at`
- `is_shared`
- `last_published_at`
- `admin_hidden`
- `created_at`
- `updated_at`

### 4.2 profiles

保存用户公开资料和治理状态。

关键字段：

- `id`
- `email`
- `nickname`
- `bio`
- `share_visibility`
- `square_hidden`
- `share_banned`
- `admin_note`
- `created_at`
- `updated_at`

### 4.3 stock_publications

保存股票发布版本。

关键字段：

- `stock_id`
- `publisher_id`
- `version_number`
- `change_note`
- `payload`
- `created_at`

### 4.4 stock_subscriptions

保存订阅关系。

关键字段：

- `subscriber_id`
- `publisher_id`
- `source_stock_id`
- `target_stock_id`
- `status`
- `last_seen_published_at`

### 4.5 follows

保存关注关系。

关键字段：

- `follower_id`
- `publisher_id`
- `created_at`

### 4.6 admin_users

保存管理员 GitHub 用户名白名单。

### 4.7 moderation_actions

保存管理员治理操作日志。

## 5. 关键业务流程

### 5.1 新用户进入系统

1. 用户通过 GitHub 或邮箱 Magic Link 登录。
2. 系统解析并保存 Supabase session。
3. 系统查询 profile。
4. 若 profile 不存在，要求设置唯一昵称。
5. 昵称创建成功后加载管理员标记、关注列表、分享邀请和我的股池。

### 5.2 新增股票并测算

1. 用户点击“新增股票”。
2. 在抽屉中搜索股票。
3. 系统匹配股票并获取市值。
4. 用户填写前瞻利润、PE、净现金、股东回报等假设。
5. 系统实时计算前瞻市值、upside、Forward PE 和股东回报率。
6. 用户保存，系统写入 `stock_records`。

### 5.3 发布股票

1. 用户编辑自己的股票。
2. 填写发布变更说明。
3. 点击发布。
4. 系统保存最新 payload。
5. 系统将股票标记为已分享。
6. 系统写入一条新的 `stock_publications` 版本记录。

### 5.4 关注并订阅他人股票

1. 用户通过分享链接或广场进入发布者股池。
2. 用户关注发布者。
3. 用户查看发布者已发布股票。
4. 用户选择复制或订阅。
5. 复制会生成静态本地记录。
6. 订阅会生成本地记录和 `stock_subscriptions` 关系。
7. 发布者后续更新时，订阅者股池显示更新提示。

### 5.5 管理员治理

1. 管理员进入管理后台。
2. 选择用户并查看其股池。
3. 根据需要隐藏用户、禁用分享、隐藏股票或写备注。
4. 系统更新相关状态，并写入 `moderation_actions`。

## 6. 非功能需求

- **可用性**：核心操作失败时需展示明确错误，不能静默失败。
- **安全性**：所有多用户数据访问必须依赖 RLS，前端判断只用于体验优化。
- **兼容性**：移动端需保持股池表格可读可操作。
- **容错性**：行情、汇率接口不可用时，应使用备用数据或固定汇率。
- **可审计性**：管理员治理操作需记录日志。
- **数据一致性**：发布、订阅、复制等动作需保持 `payload` 与结构化列一致。
